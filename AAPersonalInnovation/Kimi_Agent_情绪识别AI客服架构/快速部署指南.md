# 🚀 AI客服系统 - 完整部署指南

## 📋 系统要求

- **Python**: 3.10+ (当前检测到 3.13.9)

- **Redis**: 6.0+ (可选)

## 📦 部署步骤

### 步骤 1: 安装Python依赖

打开终端，执行以下命令：

```bash

# 安装核心依赖
pip install fastapi uvicorn sqlalchemy pymysql pydantic pydantic-settings
pip install redis python-dotenv loguru httpx

# 安装AI相关依赖（可选，需要使用LLM时）
pip install langchain langchain-openai openai
pip install chromadb rank-bm25
```

**使用清华镜像源（更快）：**
```bash

```

## 📊 数据持久化说明

### MySQL存储结构

#### conversations（会话表）
- 存储会话基本信息
- 按创建时间索引
- 支持按时间范围查询

#### messages（消息表）
- 存储所有对话消息
- **按时间顺序存储**（timestamp字段）
- 包含情绪分析、意图识别结果
- 按会话ID和时间戳索引

#### knowledge_documents（知识库表）
- 存储知识文档
- 支持软删除
- 按分类和时间索引

### 数据流程

```
用户发送消息
  ↓
1. 立即保存到MySQL（永久存储）
2. 缓存到Redis（临时，1小时）
3. 定期摘要（每15条或10分钟）
  ↓
前端查询时从MySQL读取
  ↓
支持按时间范围筛选
```

## ⚠️ 常见问题

### 1. pymysql安装失败

**解决方案：**
```bash
# 使用conda安装
conda install pymysql

# 或使用清华镜像
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple pymysql
```

### 2. MySQL连接失败

**检查：**
- MySQL是否已启动？
- 用户名密码是否正确（root/123456）？
- MySQL是否运行在localhost:3306？

**解决方案：**
```bash
# 检查MySQL状态
sudo systemctl status mysql

# 启动MySQL
sudo systemctl start mysql

# 测试连接
mysql -uroot -p123456
```

### 3. 端口8000被占用

**解决方案：**
```bash
# 查看占用端口的进程
lsof -i:8000

# 杀死进程
kill -9 [PID]

# 或者修改端口
# 编辑.env文件，修改API_PORT=8001
```

### 4. 前端页面无法访问

**检查：**
- API服务是否已启动？
- 浏览器地址是否正确？

**解决方案：**
```bash
# 确认API服务运行中
curl http://localhost:8000/health

# 如果返回{"status":"healthy"...}，说明API正常
```
