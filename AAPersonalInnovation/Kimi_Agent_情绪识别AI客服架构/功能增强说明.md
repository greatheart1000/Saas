# 功能增强说明

本文档说明AI客服系统的三个核心增强功能。

## 1. 🪟 上下文窗口管理

### 功能说明
系统现在可以为不同任务配置不同的上下文窗口大小，避免token浪费并提高响应速度。

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `CONTEXT_SENTIMENT_LIMIT` | 5 | 情绪分析时查看最近5条消息 |
| `CONTEXT_INTENT_LIMIT` | 5 | 意图识别时查看最近5条消息 |
| `CONTEXT_RESPONSE_LIMIT` | 10 | 生成回复时查看最近10条消息 |
| `CONTEXT_SUMMARY_LIMIT` | 20 | 生成总结时查看最近20条消息 |
| `CONTEXT_MAX_TOKENS` | 4000 | 最大上下文token数 |

### 使用方式

#### 方式1：全局配置（.env文件）
```env
CONTEXT_SENTIMENT_LIMIT=10
CONTEXT_RESPONSE_LIMIT=15
```

#### 方式2：API请求时指定
```python
# 为特定请求设置上下文窗口
response = await client.send_message(
    message="我想退款",
    session_id="user_001",
    context_limit=20  # 这次请求使用最近20条消息作为上下文
)
```

### 原理说明

系统会根据不同任务自动使用合适的上下文窗口：

```python
# 情绪分析只需要最近5条
sentiment = await llm_service.detect_sentiment(
    message=message,
    history=history,
    context_limit=5  # 自动使用配置的默认值
)

# 生成回复需要更多上下文
response = await llm_service.generate_response(
    query=query,
    context=context,
    history=history,
    context_limit=10  # 查看更多历史
)
```

## 2. 📝 自动摘要功能

### 功能说明
系统会自动在以下情况触发对话摘要，压缩短期记忆并保存到长期记忆：

### 触发条件

#### 条件1：消息数达到阈值
```env
SUMMARY_TRIGGER_MESSAGE_COUNT=15  # 对话达到15条消息时自动摘要
```

#### 条件2：时间间隔
```env
SUMMARY_TRIGGER_TIME_MINUTES=10  # 每10分钟自动摘要一次
```

### 摘要策略

自动摘要后会执行以下操作：

1. **生成摘要** - 使用LLM总结对话内容
2. **提取关键信息** - 订单号、手机号、关键诉求
3. **压缩短期记忆** - 只保留最近N条消息（可配置）
4. **保存到长期记忆** - 7天有效期

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `SUMMARY_AUTO_SUMMARIZE` | true | 是否启用自动摘要 |
| `SUMMARY_PRESERVE_KEY_MESSAGES` | 3 | 摘要后保留最近3条消息 |
| `MEMORY_LONG_TERM_LIMIT` | 100 | 长期记忆保留关键事实数 |

### 工作流程

```
用户发送消息
  ↓
更新短期记忆
  ↓
检查是否需要摘要：
  ├─ 消息数 >= 15？ ✅
  ├─ 时间间隔 >= 10分钟？ ✅
  └─ 从未摘要过且消息数 >= 5？ ✅
  ↓
触发自动摘要
  ├─ LLM生成摘要（100字以内）
  ├─ 提取关键事实（订单号、手机号等）
  ├─ 分析用户意图
  └─ 分析情绪趋势
  ↓
保存到长期记忆（7天）
  ↓
压缩短期记忆（只保留最近3条）
  ↓
记录摘要时间
```

### 摘要示例

**对话历史：**
```
用户: 你好
AI: 您好！有什么可以帮您？
用户: 我想查询订单
AI: 好的，请提供订单号
用户: 订单号是12345
AI: 好的，正在查询...
...（15条消息后）
```

**自动生成摘要：**
```
用户主要咨询订单#12345的物流情况。关键信息：订单号12345。
用户意图：物流查询。情绪趋势：情绪平稳。
```

### 禁用自动摘要

如果不需要自动摘要，可以在配置文件中禁用：

```env
SUMMARY_AUTO_SUMMARIZE=false
```

## 3. 🎭 Persona拟人化系统

### 功能说明
系统提供5种不同的客服人格，让AI的回复更贴近真实人类，提升用户体验。

### 可用Persona类型

#### 1. Friendly - 亲切友好型（默认）
```json
{
  "name": "亲切小助手",
  "description": "温和友善，充满同理心，让人感觉如沐春风",
  "tone_keywords": ["温暖", "亲切", "贴心", "关怀"],
  "emoji_usage": "minimal",
  "example": "您好呀！我很乐意帮您处理这个问题~ 😊"
}
```

#### 2. Cute - 可爱热情型
```json
{
  "name": "元气少女",
  "description": "活泼可爱，充满活力，用轻松愉快的语气服务",
  "tone_keywords": ["元气", "活泼", "热情", "可爱"],
  "emoji_usage": "heavy",
  "example": "好哒~ 包在我身上！一定帮您搞定！✨"
}
```

#### 3. Mature - 成熟冷静型
```json
{
  "name": "稳重的顾问",
  "description": "成熟稳重，冷静专业，值得信赖",
  "tone_keywords": ["稳重", "专业", "可靠", "理性"],
  "emoji_usage": "none",
  "example": "根据您描述的情况，建议您采取以下步骤..."
}
```

#### 4. Professional - 专业严谨型
```json
{
  "name": "专家顾问",
  "description": "严谨专业，注重效率，以解决问题为导向",
  "tone_keywords": ["专业", "高效", "严谨", "规范"],
  "emoji_usage": "none",
  "example": "针对您的问题，解决方案如下：1. ... 2. ..."
}
```

#### 5. Humorous - 幽默风趣型
```json
{
  "name": "幽默达人",
  "description": "风趣幽默，轻松愉快，用幽默化解紧张",
  "tone_keywords": ["幽默", "风趣", "轻松", "乐观"],
  "emoji_usage": "moderate",
  "example": "这个问题就像打地鼠一样，我来帮您彻底解决！"
}
```

### 使用方式

#### 方式1：全局配置（.env文件）
```env
DEFAULT_PERSONA=cute  # 默认使用可爱热情型
```

#### 方式2：API请求时指定
```python
# 为特定会话设置Persona
response = await client.send_message(
    message="我要退款",
    session_id="user_001",
    persona="cute"  # 这次使用可爱型
)
```

#### 方式3：会话级别设置
```python
# 为会话设置Persona（24小时有效）
await client.set_persona(
    session_id="user_001",
    persona="mature"
)

# 后续该会话的所有回复都会使用成熟冷静型
```

### API接口

#### 获取所有Persona
```bash
curl http://localhost:8000/api/v1/chat/personas
```

**响应示例：**
```json
{
  "personas": [
    {
      "type": "friendly",
      "name": "亲切小助手",
      "description": "温和友善，充满同理心...",
      "response_style": "温暖友好，同理心强"
    },
    {
      "type": "cute",
      "name": "元气少女",
      "description": "活泼可爱，充满活力...",
      "response_style": "活泼可爱，热情洋溢"
    }
    // ... 其他Persona
  ],
  "default": "friendly",
  "enabled": true
}
```

#### 设置会话Persona
```bash
curl -X POST http://localhost:8000/api/v1/chat/set-persona \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "user_001",
    "persona": "cute"
  }'
```

#### 获取会话Persona
```bash
curl http://localhost:8000/api/v1/chat/persona/user_001
```

## 最佳实践建议

### 1. 上下文窗口配置建议

**场景1：简单问答客服**
```env
CONTEXT_SENTIMENT_LIMIT=3      # 情绪分析不需要太多历史
CONTEXT_RESPONSE_LIMIT=5       # 简单问答，5条足够
```

**场景2：复杂售后客服**
```env
CONTEXT_SENTIMENT_LIMIT=10     # 需要看更多历史判断情绪
CONTEXT_RESPONSE_LIMIT=20      # 复杂问题需要更多上下文
```

### 2. 自动摘要配置建议

**高频对话场景（在线客服）**
```env
SUMMARY_TRIGGER_MESSAGE_COUNT=10   # 对话密集，更早摘要
SUMMARY_PRESERVE_KEY_MESSAGES=2    # 只保留最近2条
```

**低频对话场景（邮件客服）**
```env
SUMMARY_TRIGGER_MESSAGE_COUNT=30   # 对话稀疏，延迟摘要
SUMMARY_PRESERVE_KEY_MESSAGES=5    # 保留更多上下文
```

### 3. Persona选择建议

| 业务类型 | 推荐Persona | 理由 |
|---------|------------|------|
| 电商客服 | friendly | 亲切友好，提升用户满意度 |
| 年轻品牌 | cute | 活力可爱，符合品牌调性 |
| 金融/法律 | mature | 稳重专业，建立信任 |
| 技术支持 | professional | 高效准确，快速解决问题 |
| 娱乐/游戏 | humorous | 轻松愉快，提升互动乐趣 |
