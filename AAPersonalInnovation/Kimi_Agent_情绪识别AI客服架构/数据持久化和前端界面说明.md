### 1. MySQL数据持久化

**存储位置：** MySQL数据库（您指定的配置）
- 主机: localhost:3306

- 用户名: root
- 密码: 123456

**数据表结构：**

#### conversations（会话表）
```sql
CREATE TABLE conversations (
    id VARCHAR(64) PRIMARY KEY COMMENT '会话ID',
    user_id VARCHAR(64) COMMENT '用户ID',
    created_at DATETIME COMMENT '创建时间',
    updated_at DATETIME COMMENT '更新时间',
    message_count INT COMMENT '消息数量',
    status VARCHAR(20) COMMENT '状态',
    persona VARCHAR(20) COMMENT '使用的Persona',
    summary TEXT COMMENT '对话摘要',
    key_facts JSON COMMENT '关键信息',
    sentiment_trend VARCHAR(50) COMMENT '情绪趋势',
    handoff_reason TEXT COMMENT '转人工原因',
    INDEX idx_user_created (user_id, created_at),
    INDEX idx_created (created_at)
);
```

#### messages（消息表）
```sql
CREATE TABLE messages (
    id VARCHAR(64) PRIMARY KEY COMMENT '消息ID',
    conversation_id VARCHAR(64) COMMENT '会话ID',
    role VARCHAR(20) COMMENT '角色: user, assistant',
    content TEXT COMMENT '消息内容',
    timestamp DATETIME COMMENT '消息时间戳',
    created_at DATETIME COMMENT '创建时间',
    sentiment_score FLOAT COMMENT '情绪分数0-1',
    sentiment_level VARCHAR(20) COMMENT '情绪等级',
    intent_type VARCHAR(50) COMMENT '意图类型',
    router_decision VARCHAR(20) COMMENT '路由决策',
    response_time FLOAT COMMENT '响应时间(秒)',
    INDEX idx_conversation_timestamp (conversation_id, timestamp),
    INDEX idx_timestamp (timestamp)
);
```

**存储顺序：** 按时间顺序（timestamp字段）自动排序

## 🚀 快速开始

### 1. 安装依赖

```bash
cd backend
pip install pymysql aiomysql
```

### 2. 初始化数据库

```bash
python init_database.py
```

**这将执行：**
- ✅ 创建数据库 `kimi_cs`
- ✅ 创建数据表（conversations, messages, knowledge_documents）
- ✅ 插入10条示例知识文档

### 3. 更新配置文件

```bash
cp .env.example .env
```

编辑 `.env` 文件，添加MySQL配置：

```env
DATABASE_URL=mysql+aiomysql://root:123456@localhost:3306/kimi_cs
DATABASE_ECHO=false
```

### 4. 启动API服务

```bash
python -m app.api.main
```

### 5. 访问前端界面

**对话监控：**
```
http://localhost:8000/static/conversations.html
```

**知识库管理：**
```
http://localhost:8000/static/knowledge.html
```

## 📊 API接口

### 对话监控相关

#### 获取统计数据
```bash
# 今天的统计
curl http://localhost:8000/api/v1/monitor/stats?time_range=day

# 最近7天的统计
curl http://localhost:8000/api/v1/monitor/stats?time_range=week

# 最近30天的统计
curl http://localhost:8000/api/v1/monitor/stats?time_range=month
```

#### 获取会话列表
```bash
curl "http://localhost:8000/api/v1/monitor/sessions?time_range=week&limit=20"
```

#### 获取会话详情
```bash
curl http://localhost:8000/api/v1/monitor/session/session_id_here
```

### 知识库管理相关

#### 获取知识列表
```bash
curl http://localhost:8000/api/v1/knowledge/list
```

#### 添加知识
```bash
curl -X POST http://localhost:8000/api/v1/knowledge/add \
  -H "Content-Type: application/json" \
  -d '{
    "title": "测试知识",
    "content": "这是测试内容",
    "category": "订单",
    "tags": ["测试", "示例"]
  }'
```

#### 更新知识
```bash
curl -X PUT http://localhost:8000/api/v1/knowledge/update/doc_id \
  -H "Content-Type: application/json" \
  -d '{
    "title": "更新后的标题",
    "content": "更新后的内容"
  }'
```

#### 删除知识
```bash
curl -X DELETE http://localhost:8000/api/v1/knowledge/delete/doc_id
```

#### 搜索知识
```bash
curl "http://localhost:8000/api/v1/knowledge/search?keyword=退款&category=退款"
```

## 📁 数据存储说明

### Redis（缓存）- 临时存储
- **用途：** 短期记忆、会话缓存
- **有效期：** 1小时
- **不用于：** 永久化存储

### MySQL（持久化）- 永久存储 ✅
- **用途：** 所有对话历史、知识文档
- **存储方式：** 按时间顺序
- **保留时间：** 永久（除非手动删除）

### 数据流程

```
用户发送消息
  ↓
1. 保存到MySQL（立即，永久存储）
  ├─ conversations表（会话信息）
  └─ messages表（消息内容）
  ↓
2. 缓存到Redis（临时，快速访问）
  └─ 短期记忆（1小时）
  ↓
3. 定期摘要
  └─ 长期记忆保存到conversations表
```
